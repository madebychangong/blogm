# 📝 TXT 파일 기반 블로그 원고 최적화 가이드

엑셀 대신 **간단한 txt 파일**로 블로그 원고를 최적화할 수 있습니다!

---

## 🚀 빠른 시작

### 1. 대화형 모드 (제일 쉬움!)

```bash
python3 optimize_txt.py
```

그러면 메뉴가 나옵니다:
```
🎯 블로그 원고 최적화 시스템 (TXT 모드)

모드를 선택하세요:
1. 단일 파일 최적화
2. 폴더 일괄 처리
3. 종료

선택 (1-3): 1

입력 파일 경로: 내원고.txt
출력 파일 경로 (Enter=자동):
키워드 (Enter=자동 추출): 팔꿈치 쿠션 보호대
브랜드명 (선택사항): 초록입환
```

**Enter 누르면 자동!**

## 💻 커맨드라인 모드

### 기본 사용법

```bash
# 1. 가장 간단 (키워드 자동 추출)
python3 optimize_txt.py --input 원고.txt

# 2. 출력 파일 지정
python3 optimize_txt.py --input 원고.txt --output 최적화_원고.txt

# 3. 키워드 지정
python3 optimize_txt.py --input 원고.txt --keyword "팔꿈치 쿠션 보호대"

# 4. 브랜드 포함
python3 optimize_txt.py --input 원고.txt --keyword "팔꿈치 쿠션 보호대" --brand "초록입환"

# 5. 목표 글자수 변경 (기본 2000자)
python3 optimize_txt.py --input 원고.txt --target-chars 2500
```

### 약어 사용

```bash
# -i: input
# -o: output
# -k: keyword
# -b: brand
# -t: target-chars

python3 optimize_txt.py -i 원고.txt -k "팔꿈치 쿠션 보호대" -b "초록입환"
```

## 📁 폴더 일괄 처리

**여러 txt 파일을 한 번에 처리!**

```bash
# 기본 (출력 폴더 자동 생성)
python3 optimize_txt.py --folder ./원고폴더

# 출력 폴더 지정
python3 optimize_txt.py --folder ./원고폴더 --output ./결과폴더
```

**예시:**
```
원고폴더/
  ├── 원고1.txt
  ├── 원고2.txt
  └── 원고3.txt

↓ 실행

원고폴더/최적화/
  ├── 원고1_최적화.txt
  ├── 원고2_최적화.txt
  └── 원고3_최적화.txt
```

## 📄 입력 파일 형식

### 기본 형식

```
# 팔꿈치 쿠션 보호대 관련해서 사용해보신 분 계신가요?

팔꿈치 쿠션 보호대를 최근에 알게 되어서 정말 고민이 많습니다.
사실 저는 무릎 통증으로 고생하고 있는 50대인데요...

(나머지 내용)
```

### 키워드 자동 추출 규칙

1. **# 제목에서 추출**
   ```
   # 팔꿈치 쿠션 보호대 관련해서...
   → 키워드: "팔꿈치 쿠션 보호대"
   ```

2. **자동으로 불필요한 단어 제거**
   - "관련해서", "에 대해", "사용", "후기", "정보" 등은 자동 제거

3. **추출 실패 시**
   - 수동 입력 요청

## 🔧 옵션 상세 설명

| 옵션 | 약어 | 기본값 | 설명 |
|------|------|--------|------|
| `--input` | `-i` | 없음 | 입력 txt 파일 경로 |
| `--output` | `-o` | 자동 | 출력 txt 파일 경로 |
| `--folder` | `-f` | 없음 | 폴더 일괄 처리 |
| `--keyword` | `-k` | 자동 | 통키워드 |
| `--brand` | `-b` | "" | 브랜드명 |
| `--target-chars` | `-t` | 2000 | 목표 글자수 |
| `--target-keyword` | 없음 | 6 | 목표 통키워드 반복수 |

## ❓ FAQ

### Q1: 키워드가 자동으로 안 잡혀요!

**A:** 수동으로 지정하세요:
```bash
python3 optimize_txt.py --input 원고.txt --keyword "원하는 키워드"
```

### Q2: 출력 파일을 다른 이름으로 하고 싶어요!

**A:** `--output` 옵션 사용:
```bash
python3 optimize_txt.py --input 원고.txt --output 내가_원하는_이름.txt
```

### Q3: 글자수를 더 늘리고 싶어요!

**A:** `--target-chars` 옵션 사용:
```bash
python3 optimize_txt.py --input 원고.txt --target-chars 3000
```

### Q4: 폴더에 있는 파일들을 한 번에 처리하고 싶어요!

**A:** `--folder` 옵션 사용:
```bash
python3 optimize_txt.py --folder 내원고폴더
```

### Q5: 대화형 모드가 더 편한데...

**A:** 그냥 실행하세요:
```bash
python3 optimize_txt.py
```
메뉴가 나와서 단계별로 안내합니다!

## 🆚 엑셀 vs TXT 비교

| 항목 | 엑셀 모드 | TXT 모드 |
|------|----------|----------|
| 편의성 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 일괄 처리 | ✅ (한 파일에 여러 행) | ✅ (여러 파일) |
| 키워드 지정 | 엑셀 컬럼 | 자동 추출 또는 수동 |
| 결과 확인 | 엑셀에서 확인 | txt 파일로 확인 |
| 복사 편의성 | 보통 | 매우 쉬움 (txt 열어서 복사) |
| 추천 대상 | 대량 작업 | 단일/소량 작업 |

## ⚙️ 고급 설정

### 커스터마이징 가능한 항목

`optimize_txt.py`의 `optimize_single_file()` 함수 파라미터:

- `target_char_count`: 목표 글자수 (기본 2000)
- `target_whole_keyword`: 목표 통키워드 반복 (기본 6)
- `target_subkeyword_count`: 목표 서브키워드 (기본 18)
- `target_keyword_start`: 키워드 시작 문장 (기본 4)

## 📞 문제 해결

### 오류: ModuleNotFoundError

```bash
pip install -r requirements.txt
```

### 오류: 파일을 찾을 수 없습니다

경로를 절대경로로 지정:
```bash
python3 optimize_txt.py --input /full/path/to/원고.txt
```

### 오류: UnicodeDecodeError

파일을 UTF-8로 저장하세요.
